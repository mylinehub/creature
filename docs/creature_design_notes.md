# 🎭 Creature Design Notes

MathLab MYLINEHUB Mascot (M Creature)

---

# 1. 🧠 Identity

The mascot is based on:

**Capital letter "M" (MYLINEHUB)**

But it is NOT typography.

It is:

* a character
* a teacher
* a guide
* a thinking entity

---

# 2. 🧩 Core Design Idea

The shape **M** represents:

* structure
* stability
* system thinking
* engineering mindset

We convert this static symbol into:

**a living mathematical teacher**

---

* torso

# 9. 📦 Interaction with Props

The mascot must:

* point at objects
* guide attention
* stand beside visuals
* avoid overlapping confusion

---

## Supported Props

* math board
* axis plane
* formula card
* pointer stick

---

# 10. 🧑‍🏫 Teaching Role

The mascot is NOT decorative.

It is:

**a teaching interface**

---

## Responsibilities

* guide attention
* reduce confusion
* direct flow
* reinforce concepts

---

# 11. 📐 Spatial Role in Scenes

## Standard Layout

| Region | Purpose           |
| ------ | ----------------- |
| Left   | Mascot            |
| Right  | Math content      |
| Top    | Titles            |
| Bottom | secondary context |

---

## Example Layout

```
[ Mascot ]        [ Math Object ]
     ↓                 ↓
 gestures        explanation area
```

---

## Rules

* mascot should NOT block content
* content must remain primary
* mascot supports, not dominates

---

# 12. 🔗 Visual Hierarchy

Priority order:

1. Math object
2. Motion (animation)
3. Mascot gesture
4. Background

---

# 13. ⚖️ Proportional System

* mascot size consistent across scenes
* limbs scale with body
* props scale relative to mascot

---

## Example

| Element    | Scale   |
| ---------- | ------- |
| Creature   | 1.0     |
| Board      | 0.6–0.8 |
| Axis plane | 0.3–0.5 |

---

# 14. 🎯 Anchor System Importance

All placement is anchor-based:

* eyes → face anchors
* arms → shoulder anchors
* legs → hip anchors
* props → hand anchors

---

## Why

* consistency
* animation stability
* reuse across scenes

---

# 15. ⚠️ Design Constraints

| Constraint            | Reason              |
| --------------------- | ------------------- |
| No complex shapes     | animation stability |
| No random offsets     | consistency         |
| No manual positioning | scalability         |
| No heavy detail       | readability         |

---

# 16. 🧠 Behavioral Rules

* always look at active object
* gesture before explanation
* pause after motion
* avoid unnecessary motion

---

# 17. 🔄 Scene Behavior Pattern

Standard teaching flow:

1. Introduce
2. Show
3. Highlight
4. Hold

---

# 18. 📊 Consistency Checklist

Before finalizing any scene:

* ✔ alignment correct
* ✔ mascot not blocking
* ✔ gestures readable
* ✔ timing balanced
* ✔ scale consistent
